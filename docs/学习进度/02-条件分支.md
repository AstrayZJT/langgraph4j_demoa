# 条件分支

这一节要学的是：**图不再只走一条直线，而是会根据 state 自己选择下一条路。**

这就是 LangGraph4j 开始变得“像工作流引擎”的地方。

---

- [ConditionalState.java](../../src/main/java/com/example/langgraph4jdemo/conditional/ConditionalState.java)

- [ParityRouterNode.java](../../src/main/java/com/example/langgraph4jdemo/conditional/ParityRouterNode.java)

## 2. 这一步想解决什么

最小图里，执行顺序是固定的：

```text
START -> greeter -> responder -> END
```

但真实场景里，流程通常不是一条直线，而是要“分路”：

- 输入是偶数，走一条路
- 输入是奇数，走另一条路
- 检查结果够不够，不够就返回重试
- 判断是否需要检索、是否需要人工确认

这类问题就需要条件分支。

---

## 3. 这个示例的图长什么样

这次我们做的是一个“按数字奇偶分支”的小图。

```text
START -> router -> even -> END
                 \-> odd  -> END
```

更准确地说：

- `router` 先执行
- 执行完以后，条件边检查 `number`
- 如果是偶数，走 `even`
- 如果是奇数，走 `odd`

---

## 4. State 里多了什么

[`ConditionalState`](../../src/main/java/com/example/langgraph4jdemo/conditional/ConditionalState.java) 里有两个字段：

```java
public static final String NUMBER_KEY = "number";
public static final String MESSAGES_KEY = "messages";
```

### `number`

这是一个基础字段，用来存当前输入的数字。

```java
NUMBER_KEY, Channels.base(() -> 0)
```

这表示它是“普通值”，不是列表追加。

### `messages`

还是用追加型通道：

```java
MESSAGES_KEY, Channels.appender(ArrayList::new)
```

这样每个节点执行后都可以往里加一句话。

---

## 5. 图是怎么建的

核心代码在 [`ConditionalBranchService`](../../src/main/java/com/example/langgraph4jdemo/conditional/ConditionalBranchService.java)：

```java
var stateGraph = new StateGraph<>(ConditionalState.SCHEMA, ConditionalState::new)
        .addNode("router", node_async(parityRouterNode))
        .addNode("even", node_async(evenBranchNode))
        .addNode("odd", node_async(oddBranchNode))
        .addEdge(START, "router")
        .addConditionalEdges(
                "router",
                edge_async(state -> CompletableFuture.completedFuture(
                        state.number() % 2 == 0 ? "even" : "odd"
                )),
                Map.of("even", "even", "odd", "odd")
        )
        .addEdge("even", END)
        .addEdge("odd", END);
```

### 这里最关键的是 `addConditionalEdges()`

它的意思是：

1. 先从某个节点出发
2. 再执行一个“路由函数”
3. 路由函数返回一个字符串
4. 这个字符串决定下一步走哪个节点

在这个例子里：

- 偶数返回 `"even"`
- 奇数返回 `"odd"`

然后通过映射表把这个字符串接到真正的节点上。

### 到底是谁在判断条件

这里的“判断”不是 LangGraph4j 内部自动帮你猜出来的，而是你自己写的路由函数在判断。

看这一段：

```java
edge_async(state -> state.number() % 2 == 0 ? "even" : "odd")
```

这句代码的意思是：

- 如果 `number` 是偶数，返回 `"even"`
- 如果 `number` 是奇数，返回 `"odd"`

然后 LangGraph4j 负责做下一步：

- 读你返回的字符串
- 去映射表里找对应节点
- 跳到那个节点继续执行

所以这里的职责分工很清楚：

- **你**负责写条件判断
- **LangGraph4j**负责按结果跳路

## 6. `router` 节点做什么

[`ParityRouterNode`](../../src/main/java/com/example/langgraph4jdemo/conditional/ParityRouterNode.java) 只负责打印当前数字，并写一条日志消息：

```java
return Map.of(ConditionalState.MESSAGES_KEY, "Router saw number " + state.number());
```

它的作用更像“检查点”。

真正决定下一步的是条件边，不是这个节点本身。

---

## 7. `even` / `odd` 节点做什么

### 偶数分支

[`EvenBranchNode`](../../src/main/java/com/example/langgraph4jdemo/conditional/EvenBranchNode.java) 会写：

```java
return Map.of(ConditionalState.MESSAGES_KEY, state.number() + " is even");
```

### 奇数分支

[`OddBranchNode`](../../src/main/java/com/example/langgraph4jdemo/conditional/OddBranchNode.java) 会写：

```java
return Map.of(ConditionalState.MESSAGES_KEY, state.number() + " is odd");
```

这就能很直观地看出：同一张图，输入不同，路径就不同。

---

## 8. 为什么这一步重要

因为你终于开始看到 LangGraph4j 的核心价值了：

- 图不是死的线性调用
- 图可以按状态动态分支
- 图的下一步不是写死在代码里，而是由状态决定

这比单纯的“顺序调用函数”更像真正的 agent workflow。

---

## 9. 程序是怎么跑这个示例的

`LangGraph4jDemoApplication` 启动后，会先跑最小图，再跑条件分支图。

条件分支这一步会跑两次：

- `4` -> 偶数分支
- `7` -> 奇数分支

这样你能在一次启动里同时看到两条路。

---

## 10. 怎么验证这一步成功

你在控制台里应该看到这两段：

### 4 的时候

- `Conditional branch demo start: 4`
- `ParityRouterNode executing. Current number: 4`
- `EvenBranchNode executing...`
- `NodeOutput{ node=even ... }`
- `Conditional branch demo end: 4`

### 7 的时候

- `Conditional branch demo start: 7`
- `ParityRouterNode executing. Current number: 7`
- `OddBranchNode executing...`
- `NodeOutput{ node=odd ... }`
- `Conditional branch demo end: 7`

只要这两次都按预期跑出来，就说明条件分支已经通了。

---

## 11. 这一节学完后你应该明白什么

你应该真正理解这些事：

1. 条件分支不是“节点自己判断下一步”
2. 而是“边根据 state 路由下一步”
3. `addConditionalEdges()` 是这类能力的入口
4. 同一张图可以根据不同输入走不同路径
