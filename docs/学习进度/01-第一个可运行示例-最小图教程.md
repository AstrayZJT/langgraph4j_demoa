# 第一个可运行示例：最小图教程

这一节是 LangGraph4j 的起点。

它的目标不是做业务，而是先把最小闭环跑通：

- 定义 `State`
- 定义 `Node`
- 定义 `Edge`
- `compile()`
- `stream()` 跑起来

如果这一步你真的看懂了，后面条件分支、循环、checkpoint、子图，都会顺很多。

## 1. 这一节对应哪些代码

- [LangGraph4jDemoApplication.java](../../src/main/java/com/example/langgraph4jdemo/LangGraph4jDemoApplication.java)
- [SimpleGraphService.java](../../src/main/java/com/example/langgraph4jdemo/minimal/SimpleGraphService.java)
- [SimpleState.java](../../src/main/java/com/example/langgraph4jdemo/minimal/SimpleState.java)
- [GreeterNode.java](../../src/main/java/com/example/langgraph4jdemo/minimal/GreeterNode.java)
- [ResponderNode.java](../../src/main/java/com/example/langgraph4jdemo/minimal/ResponderNode.java)

你可以把这 5 个文件理解成：

- 主类负责启动
- service 负责建图和执行
- state 负责装数据
- node 负责做一步事情

---

## 2. 这段程序到底在干嘛

启动后，它会自动创建一张最简单的图：

```text
START -> greeter -> responder -> END
```

然后给图塞一个初始状态：

```text
messages = "Let's begin!"
```

接着图会一步一步执行：

1. 进入 `START`
2. 执行 `greeter`
3. 执行 `responder`
4. 结束于 `END`

同时把每一步状态变化打印到控制台。

这就是你现在看到的输出。

---

## 3. 程序入口是怎么触发的

主类里有一个 `ApplicationRunner`：

```java
@Bean
ApplicationRunner graphRunner(SimpleGraphService simpleGraphService) {
    return args -> simpleGraphService.runOnce();
}
```

这段的意思是：

- Spring Boot 启动完成
- 容器里 Bean 都准备好之后
- 自动调用 `runOnce()`

所以你不是手动点某个按钮去跑图，而是“应用一启动，图就跑一次”。

这很适合学习阶段，因为你打开控制台就能立刻看到图的执行过程。

---

## 4. State 是什么

[`SimpleState`](../../src/main/java/com/example/langgraph4jdemo/minimal/SimpleState.java) 是这个图共享的数据结构。

它里面只有一个字段：

```java
public static final String MESSAGES_KEY = "messages";
```

然后定义 schema：

```java
public static final Map<String, Channel<?>> SCHEMA = Map.of(
        MESSAGES_KEY, Channels.appender(ArrayList::new)
);
```

### 这里的意思是什么

`state` 就是图里大家共享的一份上下文。

`messages` 用的是 `appender`，意思不是覆盖，而是追加。

所以每个节点执行完后，新的消息会往后加，最后你能看到一串累积结果。

### 为什么要这么设计

因为 LangGraph4j 的核心不是“单个函数返回一个值”，而是“流程中的状态如何一步步演化”。

这个 `messages` 就是最小版的状态演化示例。

---

## 5. 图是怎么建出来的

核心代码在 [`SimpleGraphService`](../../src/main/java/com/example/langgraph4jdemo/minimal/SimpleGraphService.java)：

```java
var stateGraph = new StateGraph<>(SimpleState.SCHEMA, SimpleState::new)
        .addNode("greeter", node_async(greeterNode))
        .addNode("responder", node_async(responderNode))
        .addEdge(START, "greeter")
        .addEdge("greeter", "responder")
        .addEdge("responder", END);

this.compiledGraph = stateGraph.compile();
```

### 先看 `StateGraph`

`StateGraph` 是“图定义”。

它还不是最终执行器，只是在描述：

- 这个图有哪些状态字段
- 这个图有哪些节点
- 节点之间怎么连

### 再看 `addNode()`

```java
.addNode("greeter", node_async(greeterNode))
.addNode("responder", node_async(responderNode))
```

这一步只是把节点放进图里。

节点放进去，不代表执行顺序已经定死。

### 真正决定顺序的是 `addEdge()`

```java
.addEdge(START, "greeter")
.addEdge("greeter", "responder")
.addEdge("responder", END);
```

这三条边真正表达的是：

```text
START -> greeter -> responder -> END
```

也就是说：

- `addNode()` 是“注册节点”
- `addEdge()` 是“规定路怎么走”

这是很多初学者第一次看 LangGraph4j 时最容易卡住的地方。

---

## 6. `var` 到底是什么

你看到的是：

```java
var stateGraph = new StateGraph<>(...)
```

这里的 `var` 不是“没有类型”，只是 Java 帮你推断类型。

它等价于：

```java
StateGraph<SimpleState> stateGraph = new StateGraph<>(...);
```

所以 `stateGraph` 真实类型还是 `StateGraph<SimpleState>`，当然就有 `compile()`。

### 为什么能链式写

因为 `addNode()` 和 `addEdge()` 这些方法返回的还是 `StateGraph` 自己。

所以可以连着写：

```java
.addNode(...)
.addEdge(...)
.compile()
```

这就是 fluent API。

---

## 7. `compile()` 是什么

```java
this.compiledGraph = stateGraph.compile();
```

这里的 `compile()` 不是 Java 编译。

Java 编译是 Maven / `javac` 做的，它负责把 `.java` 变成 `.class`。

LangGraph4j 的 `compile()` 是图层面的编译，它做的是：

- 检查图结构
- 固化节点和边
- 生成 `CompiledGraph`
- 把图从“定义阶段”变成“运行阶段”

你可以把它理解成：

- `StateGraph` = 图纸
- `CompiledGraph` = 可执行方案

---

## 8. `CompiledGraph` 之后怎么跑

真正执行时用的是：

```java
compiledGraph.stream(Map.of(SimpleState.MESSAGES_KEY, "Let's begin!"))
```

这表示：

- 以这份初始 state 开始
- 沿着图里的边走
- 一边走一边输出每一步结果

### 为什么用 `stream()`

因为它适合学习和调试。

你不仅能看到最终结果，还能看到：

- 哪个节点被执行了
- state 怎么变了
- 最后在哪结束

---

## 9. 两个 Node 分别做了什么

### GreeterNode

它读取当前 state，然后追加一条问候消息：

```java
return Map.of(SimpleState.MESSAGES_KEY, "Hello from GreeterNode!");
```

### ResponderNode

它先看看前面有没有问候消息：

```java
if (currentMessages.contains("Hello from GreeterNode!")) {
    return Map.of(SimpleState.MESSAGES_KEY, "Acknowledged greeting!");
}
```

如果有，就返回确认信息。

### 这两个节点的意义

它们虽然简单，但已经展示了最核心的模式：

- 节点能读 state
- 节点能写 state
- 下一节点能基于前一节点的结果继续做事

这就是图式工作流的最小闭环。

---

## 10. 控制台里应该看到什么

正确运行后，你应该看到这些关键输出：

- `=== LangGraph4j minimal graph start ===`
- `NodeOutput{ node=__START__ ... }`
- `GreeterNode executing...`
- `NodeOutput{ node=greeter ... }`
- `ResponderNode executing...`
- `NodeOutput{ node=responder ... }`
- `NodeOutput{ node=__END__ ... }`
- `=== LangGraph4j minimal graph end ===`

最后退出码是 `0`，说明程序正常结束。

---

## 11. 这一步学完之后你应该真正理解什么

你至少应该能回答这几个问题：

1. `State` 是什么
2. `Node` 是什么
3. `Edge` 是什么
4. 为什么要先 `compile()`
5. 为什么执行时用 `stream()`
6. 为什么执行顺序看边，不看节点写入顺序

如果这 6 个点明白了，LangGraph4j 的地基就算打好了。
