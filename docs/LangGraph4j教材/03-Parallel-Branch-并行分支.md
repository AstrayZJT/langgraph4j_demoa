# Parallel Branch：并行分支

LangGraph4j 支持把一个节点之后的多个后继节点并行执行。

这类能力适合：

- 多路分析
- 并行检索
- 多模型并发判断
- 多 agent 协同

## 基本思路

如果一个节点后面连了多个节点，那么这些后继节点可以视作并行分支。

但要注意两点：

- 并行执行需要给指定并行节点提供 `Executor`
- 如果不提供，默认可能还是按顺序执行

## RunnableConfig 的作用

并行分支的执行器通常通过 `RunnableConfig` 传入。

也就是说，并行能力不是写死在图结构里的，而是可以在运行时按需打开。

## 约束

官方文档强调了几个限制：

- 目前主要支持 Fork-Join 风格
- 一个并行步骤里只允许一层并行
- 不允许在并行段里再塞条件边

这意味着它更像“受控并发”，不是无限自由的流程图。

## 如何定义

最直接的方式是：让多个节点连到同一个前驱节点上。

然后在运行时，为那个前驱节点配置并行执行器。

## 为什么这样设计

优点很清楚：

- 能把耗时步骤拆开并发跑
- 结果最终在汇合点收敛
- 流程结构仍然清晰

它比手写 `CompletableFuture` 更像工作流，而不是零散线程代码。
