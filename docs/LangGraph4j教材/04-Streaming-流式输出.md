# Streaming：流式输出

LangGraph4j 原生支持流式执行。

这件事很重要，因为 agent 工作流通常不是“一步出结果”，而是会经历很多中间状态。

## `.stream()`

`stream()` 会返回一个可迭代的异步生成器。

你在遍历它时，会依次拿到每一步的输出，通常表现为：

- 执行了哪个节点
- 当前状态变成了什么

这就是你现在控制台里看到的 `NodeOutput`。

## 为什么流式很有价值

- 调试方便
- 能看过程，不只看结果
- 适合前端实时展示
- 适合长任务和可中断任务

## Streaming of Streaming

官方还支持“嵌套流”。

意思是：

- 你在一个节点里也可以返回一个流
- 外层 graph 的流可以继续把这个内层流一起吐出来

这对“子流程里还有子流程”的场景很友好。

## 和 LangChain4j 的流式结合

如果你用 LangChain4j 的流式模型，LangGraph4j 可以把模型 token 流和图步骤流一起组织起来。

这样你看到的不只是模型吐词，还能看到整个工作流怎么走。
