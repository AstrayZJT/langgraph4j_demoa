# Subgraphs：子图

子图就是“图里再放一个图”。

它本质上是封装和组合。

## 为什么要用子图

- 复用一组节点
- 组织多 agent 系统
- 把复杂系统拆给不同团队开发
- 让父图只关心接口，不关心内部实现

## 三种接入方式

### 1. 直接挂编译后的子图

适合父图和子图共享状态。

### 2. 用节点动作去调用子图

适合父图和子图状态结构不同，需要先做转换。

### 3. 直接挂状态子图

适合父图和子图关系非常紧密、几乎共享全部内容。

## 怎么理解

你可以把子图看成：

- 一个可复用业务模块
- 一个独立工作流单元
- 一个更小的图组件

## 为什么好用

它让大系统拆分得很干净。

你不用把所有节点都堆在一张大图里，复杂度会低很多。
