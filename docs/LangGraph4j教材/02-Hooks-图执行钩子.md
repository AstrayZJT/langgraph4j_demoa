# Hooks：图执行钩子

Hooks 的作用，是在图的节点或条件边执行前后插入自定义逻辑。

它适合做这些事：

- 记录日志
- 打点和统计
- 调试执行过程
- 在步骤之间修改状态
- 动态改变执行控制

## 两类 Hook

### NodeHook

用于拦截节点执行。

### EdgeHook

用于拦截条件边执行。普通固定边没有“函数体”，所以主要是条件边才有 hook 的意义。

## 三种时机

### BeforeCall

在节点或边真正执行前触发。

### AfterCall

在节点或边执行后触发，可以拿到本次结果。

### WrapCall

把原执行逻辑包起来。它既能在前后插逻辑，也能决定要不要继续执行原函数。

## 注册方式

Hooks 可以注册到：

- 整个图
- 指定节点
- 指定条件边

也就是说，你可以做全局埋点，也可以只盯着某一个关键节点。

## 执行顺序

同一个节点或边上挂多个 hook 时，顺序要记住：

- `BeforeCall` 和 `AfterCall` 是后加的先执行
- `WrapCall` 是先加的先包裹，外层先进入，最后退出

## 为什么有用

它把“业务逻辑”和“观测/控制逻辑”分开了。

这样图本身保持干净，但你又能在需要时加日志、监控、调试、审计。
