# Checkpoint (Postgres)：Postgres 持久化

这个模块的作用是把图的执行状态持久化到 PostgreSQL。

简单说，就是把“跑到哪一步了”存下来。

## 它解决什么问题

- 程序重启后还能继续
- 图执行中断后可以恢复
- 能查看历史状态
- 能做会话级记忆

## 主要能力

- 持久化整个 workflow state
- 内存缓存，减少数据库往返
- 自动初始化表结构
- 和 LangGraph4j 状态体系直接集成

## 依赖要求

- PostgreSQL 16.4+
- Java 17+
- LangGraph4j core

## 使用方式

先加依赖，再初始化 `PostgresSaver`。

初始化时通常要提供：

- 数据库地址
- 用户名密码
- 数据库名
- 状态序列化器

还可以控制：

- 是否先删表
- 是否自动建表

## 为什么它适合 LangGraph4j

因为 LangGraph4j 的核心就是状态流转。

一旦把状态存起来，图就从“内存里的流程”变成了“可恢复的工作流”。
